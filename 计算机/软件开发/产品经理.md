# 产品经理


## 基础介绍

沟通 + 文档 + 原型
研发做“如何做”，产品经理决定“做什么、为什么做”

1. 市场与用户调研
    - 了解用户需求
2. 产品规划
    - 设定版本规划（MVP、V1.0、V2.0…）
3. 需求管理
    - 输出需求文档（PRD, Product Requirement Document）
4. 项目协作
    - 与研发、测试、UI/UX 设计师沟通需求
    - 在开发过程中 答疑，保证需求被正确理解
5. 产品上线与迭代
    - 跟进产品上线，收集用户反馈
    - 通过埋点、数据分析优化产品
